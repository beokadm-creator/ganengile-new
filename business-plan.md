## 1. Executive Summary

### 1.1 비전

**가는길에**는 서울 지하철 이용자의 출퇴근 동선을 활용한 혁신적인 크라우드 배송 플랫폼입니다. 우리는 기존 동선을 낭비 없이 활용하여 배송 비용을 절감하고, 지하철 이용자에게는 추가 수익을 창출하는 선순환 생태계를 구축합니다.

### 1.2 핵심 가치 제안

- **비용 절감:** 기존 퀵서비스 대비 20-30% 저렴한 배송비 (3,000~8,000원)
- **시간 효율성:** 출퇴근길에 배송하여 추가 시간 투자 불필요
- **신뢰성:** AI 기반 매칭, 실시간 추적, 평점 시스템
- **사회적 가치:** 일자리 창출, 친환경적 배송 모델

### 1.3 사업 개요

| 항목 | 내용 |
|------|------|
| 서비스 명 | 가는길에 (GaneunGile) |

## 2. 비즈니스 개요

### 2.1 서비스 개념

**가는길에**는 서울 지하철을 이용하는 출퇴근족이 자신의 동선(경로)을 등록하고, 다른 사용자의 배송 요청을 수행하여 수익을 창출하는 P2P 배송 플랫폼입니다.

#### 핵심 문제 해결

1. **배송비 부담:** 기존 퀵서비스 비용이 6,000~15,000원으로 부담스러움
2. **시간 낭비:** 별도의 배송 기사가 이동하는 시간이 배송 시간의 대부분을 차지
3. **유휴 인력 활용:** 수많은 출퇴근족이 지하철로 이동하며 유휴 시간을 보냄

#### 해결 방안

- **출퇴근길 배송:** 지하철 이용자가 자신의 동선으로 배송 수행
- **AI 매칭:** 경로, 시간, 평점을 고려한 최적 매칭
- **실시간 추적:** 배송 현황을 실시간으로 추적 가능

### 2.2 미션 & 비전

#### 미션 (Mission)
> "지하철 동선을 연결하여 일상을 더 가깝게 만듭니다."

#### 비전 (Vision)
> "대한민국 1등 지하철 크라우드 배송 플랫폼"

#### 핵심 가치 (Core Values)
- **효율성 (Efficiency):** 기존 동선을 낭비 없이 활용
- **신뢰 (Trust):** 투명한 평점 시스템과 실시간 추적
- **혁신 (Innovation):** AI 기반 매칭 알고리즘
- **공유 (Sharing):** 개인의 유휴 자원을 공유하여 사회적 가치 창출

### 2.3 서비스 타겟

#### 주요 타겟 고객

| 세그먼트 | 특징 | 니즈 |
|----------|------|------|
| **개인 글러** | 2030 직장인, 대학생 | 저렴하고 빠른 배송 |
| **개인 길러** | 출퇴근하는 직장인 | 출퇴근길 부수입 |
| **사업자** | 온라인 쇼핑몰, 소상공인 | 정기적 배송, 비용 절감 |

## 3. 시장 분석

### 3.1 시장 규모

#### 대한민국 택배/퀵서비스 시장

- **2025년 시장 규모:** 약 7조원
  - 택배: 5조원 (일일 400만건)
  - 퀵서비스: 2조원 (일일 50만건)
- **성장률:** 연평균 8% 성장
- **수도권 비중:** 전체의 60% (약 4.2조원)

#### 서울 지하철 이용객

- **일일 이용객:** 약 700만명 (1~9호선)
- **출퇴근 시간대 (07-09시, 18-20시):** 약 200만명/일
- **잠재적 길러 풀:** 200만명 × 5% 참여율 = 10만명

### 3.2 시장 트렌드

#### 1. 크라우드소싱 확대
- 우버, 에어비앤비 등 플랫폼 경제 성장
- 개인 간 거래(P2P)에 대한 신뢰도 증가
- 긱 이코노미(Gig Economy) 활성화

#### 2. 배송 수요 증가
- e-commerce 성장으로 소형 배송 수요 급증
- 1인 가구 증가로 소포 배송 수요 증가
- 당일 배송에 대한 기대치 상승

#### 3. 친환경·ESG 경영
- 탄소 배출 저감에 대한 사회적 요구
- 유휴 자원 활용에 대한 긍정적 인식
- 일자리 창출을 통한 사회적 가치 창출

### 3.3 타겟 시장

#### 1차 타겟: 서울 강남/서초/마포 지역

- 이유: 소득 수준 높음, 배송 수요 많음, 지하철 이용률 높음
- 타겟 고객: 2030 직장인, 오피스텔 거주자

#### 2차 타겟: 수도권 전체

- 확장 시점: 서울 안정화 후 (1년차 말)
- 확장 지역: 경기 인천, 경기 남부, 경기 북부

#### 3차 타겟: 지방 대도시

- 확장 시점: 3년차 이후
- 대상: 부산, 대구, 대전, 광주 등 지하철 보유 도시

### 3.4 경쟁사 분석

#### 직접 경쟁사

| 경쟁사 | 장점 | 단점 | 우리의 우위 |
|--------|------|------|------------|
| **배달의민족/배민커넥트** | 브랜드 인지도, 높은 사용자 수 | 높은 배송비 (6,000~15,000원) | 20-30% 저렴 |

## 4. 서비스 설명

### 4.1 핵심 서비스

#### A. 개인 배송 서비스 (B2C)

**글러 (배송 요청자)**
1. 앱에서 배송 요청 생성 (출발역, 도착역, 물건 정보)
2. AI 기반으로 최적 길러 매칭 (최대 3명 추천)
3. 길러 선택 후 실시간 추적
4. 완료 후 평가 및 결제

**길러 (배송 대행자)**
1. 자신의 동선 등록 (최대 5개 경로)
2. 매칭 알림 수신 (FCM 푸시)
3. 수락 후 픽업 및 배송
4. 완료 후 수익 확정 (요금의 90%)

#### B. 사업자 배송 서비스 (B2B)

**사업자 (Business Gler)**
- 정기 배송 스케줄링
- API 연동 (쇼핑몰 주문 자동화)
- 월 구독 플랜 (Basic, Pro, Enterprise)
- 세금 계산서 자동 발급

**전문 길러 (B2B Giller)**
- Silver/Gold/Platinum 등급
- 우선 매칭 혜택
- 월 배송 한도 증가
- 전용 요청 할당

#### C. 위치 파트너 서비스

**위치 파트너 (Location Partner)**
- 수령 장소 제공 (편의점, 카페, 역매점)
- 수수료 수익 (건당 300원)
- 보관 시스템 (CCTV, 냉장고)
- 추가 방문 유입 (교차 판매)

### 4.2 기술 아키텍처

#### Frontend

| 기술 | 용도 |
|------|------|
| React Native | iOS/Android 크로스 플랫폼 |
| Expo SDK 54 | 개발 및 배포 도구 |
| TypeScript | 타입 안전성 |
| React Navigation | 화면 네비게이션 |

#### Backend

| 기술 | 용도 |
|------|------|
| Firebase Firestore | NoSQL 데이터베이스 |
| Firebase Auth | 사용자 인증 |
| Firebase Functions | 서버리스 백엔드 |
| FCM | 푸시 알림 |

#### 핵심 기술

1. **AI 매칭 알고리즘**
   - 경로 일치도 (40%)
   - 시간 일치도 (20%)
   - 길러 평점 (30%)
   - 완료율 (10%)

2. **실시간 추적 시스템**
   - Firebase Firestore 실시간 리스너
   - 위치 업데이트 (30초 간격)
   - 진행률 계산 (0-100%)

3. **QR 인증 시스템**
   - 픽업 코드 (6자리)
   - 완료 코드 (6자리)
   - 사진 인증 (분실/파손 방지)

### 4.3 서비스 워크플로우

#### 길러 동선 등록 플로우

```
1. 길러가 동선 등록 (출발역, 도착역, 시간, 요일)
2. Firestore에 routes 컬렉션에 저장
3. 매칭 시스템에서 경로 캐싱
4. 해당 동선과 일치하는 요청이 오면 푸시 알림
```

#### 배송 요청 및 매칭 플로우

```
1. 글러가 배송 요청 생성
2. AI 매칭 알고리즘으로 후보 길러 추천 (최대 3명)
3. 글러가 길러 선택 후 요청 수락
4. 길러에게 픽업 알림 (출발역, 시간, PIN)
5. 길러가 픽업 후 QR 인증
6. 배송 추적 시작
7. 도착 후 완료 인증
8. 평가 및 정산
```
