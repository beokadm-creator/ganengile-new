## Essential (Post-Compact)
- 가는길에는 “요청자(Requester)”와 “길러(Giller)” 두 역할로 배달/수거/출장을 매칭한다.
- 처음에는 **요청자**로 시작하고, 필요 시 **길러 신청** 절차로 역할을 확장한다.
- 계정 생성(가입)과 서비스 이용 준비(온보딩)는 분리된다.

## 1) 역할 이해하기
- **요청자**: 배송/수거/출장 요청을 만드는 사용자
- **길러**: 요청을 수행하는 사용자(심사/승급 절차가 있을 수 있음)

## 2) 기본 사용 흐름(요청자)
1. 로그인/계정 생성
2. 첫 요청 진입 시점에 “이용자 온보딩”을 완료
3. 배송/수거/출장 요청 생성
4. 매칭 결과 확인 → 진행 상황 추적 → 완료 후 리뷰/정산

## 3) 길러가 되고 싶다면
- 앱 내 “길러 신청/전환” 진입점에서 별도 절차로 진행한다.
- 자세한 제품 원칙은 `../USER-ENTRY-FLOW-STANDARD.md` 참고.
