# 문서 운영 규칙

## 목표

문서가 코드보다 늦게 갱신되어 오류를 유발하는 일을 막기 위한 규칙입니다.

## 규칙

1. 새 기능 문서는 `docs/` 아래 적절한 카테고리에만 추가합니다.
2. 루트에 분석 보고서나 임시 정리 문서를 새로 만들지 않습니다.
3. 배포 전에는 코드 변경 영역에 맞는 문서를 함께 수정합니다.
4. 과거 보고서성 문서는 유지하지 않고 삭제를 기본으로 합니다.
5. 다른 에이전트는 항상 [docs/README.md](/Users/aaron/ganengile-new/docs/README.md)부터 읽습니다.

## 문서 갱신 매핑

- `src/screens/*` 변경: `docs/flows/*`
- `src/services/*` 변경: `docs/core/*`
- `admin-web/*` 변경: `docs/admin/*`
- `firebase.json`, 배포 스크립트, 환경변수 변경: `docs/ops/*`
- 데이터 시드/보정 스크립트 변경: `docs/data/*`

## 금지 사항

- 루트에 `*_REPORT.md`, `TODO.md`, `PLAN.md`를 다시 만들지 않습니다.
- 임시 점검 결과를 새 마크다운으로 남기기 전에 기존 문서에 흡수할 수 있는지 먼저 확인합니다.
- 코드와 맞지 않는 수치나 정책을 문서에 남겨두지 않습니다.
