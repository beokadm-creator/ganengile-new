# 실서비스 스모크 테스트

## 목표

배포 후 실제 운영 흐름이 끊기지 않는지 빠르게 점검합니다.

## 사전 조건

- 요청자 계정 1개
- 길러 승인 완료 계정 1개
- 관리자 웹 접근 가능
- 운임 캐시와 역 매핑 최신화 완료

## 앱 확인 순서

1. 요청 생성
2. 채팅 진입
3. 길러 수락
4. 직접전달 또는 사물함 전달
5. 사용자 승인
6. 정산/환급 로그 확인

## 필수 체크 포인트

- 요청 생성 시 운임이 0으로 계산되지 않는가
- 가격 UI에 깨진 값이나 `0` 단독 텍스트가 없는가
- 길러 매칭 후 채팅으로 이어지는가
- 사물함 상세 위치가 표시되는가
- 완료 후 관리자 정산 로그가 생성되는가

## 관리자 확인 순서

1. `verifications`에서 길러 상태 확인
2. `deliveries`에서 배송 상태 확인
3. `settlements`에서 정산 생성 확인
4. `lockers`에서 사물함 데이터 확인

## 실패 시 우선 확인

- `config_stations` 역 코드 누락
- `config_fares` 캐시 누락
- Firestore 인덱스 누락
- 역할 상태와 승인 상태 불일치
